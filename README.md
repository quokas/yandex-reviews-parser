# Тестовое задание: Интеграция с Яндекс.Картами (Laravel 11 + Vue 3 SPA)

Веб-приложение для интеграции с Яндекс.Картами, просмотра рейтинга организации и вывода отзывов с постраничной навигацией.

## 🛠️ Стек технологий
* **Бэкенд:** PHP 8.4 + Laravel 11
* **Фронтенд:** Vue 3 (Composition API) + Tailwind CSS v4 + Vite
* **База данных:** СУБД SQLite (для легкости развертывания)
* **Авторизация:** SPA Authentication на базе cookie-сессий (Laravel Sanctum / Web Guard)

## 🧠 Архитектурные особенности и выполнение ТЗ
1. **Тонкие роуты и Eloquent модели:** Бизнес-логика хранения данных полностью разделена между реляционными моделями `Organization` и `Review` с использованием связей `HasMany` и `BelongsTo`.
2. **Пагинация без перезагрузки (Vue 3):** Реализована асинхронная постраничная навигация через Axios. Интерфейс запрашивает данные у API Laravel порциями строго по 50 отзывов, обеспечивая бесшовное переключение страниц без моргания экрана и перезагрузок DOM.
## 🧠 Обоснование технологии парсинга (Выполнение главного требования ТЗ)
Для обхода защиты Яндекс.Карт от ботов, автоматической подгрузки данных скриптом по мере прокрутки (Lazy Loading) и извлечения до ~600 отзывов был выбран метод **Headless-браузера на базе Node.js и библиотеки Puppeteer**.

* **Почему не разбор внутренних API-запросов карточки?** Внутренние AJAX-курсоры Яндекса защищены динамическими CSRF-токенами, сессионными куками и шифрованием заголовков. Попытка симулировать эти запросы напрямую через PHP приводит к моментальной блокировке со стороны SmartCaptcha или ошибкам `403 Forbidden`.
* **Преимущества Puppeteer:** Скрипт запускает изолированный безголовый Chrome, эмулируя поведение реального пользователя. Он находит контейнер отзывов и циклично прокручивает страницу (`window.scrollBy`), дожидаясь асинхронной подгрузки элементов JavaScript. Это позволяет безопасно собрать полный массив отзывов (~600 штук), точный средний рейтинг, количество оценок напрямую из DOM-разметки и кэшировать их в локальную базу данных SQLite.

4. **Обработка состояний:** В интерфейсе Vue 3 полностью прописаны реактивные состояния загрузки (`Loading...`) и вывод понятных ошибок валидации / сервера во всплывающих плашках.

## 🚀 Инструкция по локальному запуску

Поскольку проект разработан без использования тяжелого Docker-окружения, его запуск на вашем компьютере займет не более 2 минут.

### 1. Подготовка бэкенда
Установите зависимости Composer и очистите кэш конфигурации:
```bash
composer install
php artisan optimize:clear
```

### 2. Подготовка базы данных
Запустите миграции для создания таблиц в SQLite и принудительно добавьте тестового пользователя через Сидер:
```bash
php artisan migrate:fresh --seed
```

### 3. Подготовка фронтенда
Установите npm-пакеты для Vue 3 и Tailwind v4:
```bash
npm install
```

### 4. Запуск приложения
Откройте два окна терминала и запустите процессы одновременно:
* **В первом терминале (Сервер PHP):**
  ```bash
  php artisan serve
  ```
* **Во втором терминале (Сборщик Vite):**
  ```bash
  npm run dev
  ```

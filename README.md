# Сервис мониторинга и кэширования отзывов Яндекс.Карт

Полносистемное Fullstack-приложение для автоматического сбора, дедупликации и постраничного отображения отзывов организаций из Яндекс.Карт. Разработано строго в соответствии с требованиями технического задания.

## 🛠 Технологический стек
- **Backend:** Laravel 11 (API, Eloquent, Сборщики коллекций)
- **Frontend:** Vue 3 SPA (Composition API, Axios)
- **Database:** SQLite (Локальное легковесное кэширование)
- **Automation:** Node.js + Puppeteer (Скрипт глубокого веб-скроллинга)
- **Design:** Tailwind CSS v4 + Vite (Адаптивный интерфейс)

## 🎯 Архитектурные особенности и обход антифрода
1. **Human-like Эмуляция:** Парсер `parcer.cjs` имитирует реального пользователя за счёт случайных задержек, плавного скроллинга колесиком мыши (`page.mouse.wheel`) и нажатий клавиш (`PageDown`, `ArrowDown`).
2. **Защита от таймаутов:** Обмен данными между Node.js и Laravel оптимизирован под жесткие лимиты бесплатных интернет-тоннелей (ngrok/localtunnel). Скроллинг ограничен сбалансированным количеством шагов для мгновенного автоматического вывода.
3. **Умный контент-фильтр:** Встроенный парсер дерева DOM изолирует системный мусор Яндекса (аналитические плашки категорий, проценты, уровни дегустаторов, кнопки вкладок) и собирает исключительно чистые тексты отзывов.
4. **Ультимативная дедупликация:** На стороне бэкенда Laravel отзывы очищаются от динамических хвостов Яндекса (`... еще`), приводятся к нижнему регистру и сравниваются по уникальным буквенно-цифровым слепкам (хэш-ключам). Это исключает дублирование при повторной синхронизации.
5. **Безопасность БД:** Массовое сохранение в SQLite реализовано порциями (чанками) по 100 элементов, что полностью защищает оперативную память сервера в среде Windows/Linux.

## 🚀 Инструкция по локальному развертыванию

### Способ 1. Стандартный запуск

1. **Клонируйте репозиторий и перейдите в папку:**
   ```bash
   git clone <ссылка_на_ваш_репозиторий>
   cd testapp
   ```

2. **Установите зависимости бэкенда и настройте окружение:**
   ```bash
   composer install
   cp .env.example .env
   php artisan key:generate
   ```

3. **Настройте базу данных SQLite:**
   Создайте пустой файл `database/database.sqlite` (или подтвердите создание при миграции) и запустите сидеры:
   ```bash
   php artisan migrate:fresh --seed
   ```

4. **Установите зависимости фронтенда и Puppeteer:**
   ```bash
   npm install
   npm run build
   ```

5. **Запустите локальный сервер:**
   ```bash
   php artisan serve
   ```
   Проект будет доступен по адресу: `http://127.0.0.1:8000`

### Способ 2. Запуск через Docker (Контейнеризация)

Проект полностью готов к мгновенному развертыванию на любом удаленном сервере одной командой:
```bash
docker-compose up -d --build
```
